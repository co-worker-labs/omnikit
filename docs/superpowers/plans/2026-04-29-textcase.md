# Text Case Converter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a single-line text case converter at `/textcase` that simultaneously displays all 11 case formats with auto-detection and per-row copy buttons, fully client-side.

**Architecture:** A pure logic module (`libs/textcase/main.ts`) wraps `change-case@5` + `title-case@4` to define the 11 formats and a round-trip detection function. A two-file page (`page.tsx` + `textcase-page.tsx`) follows the established storageunit pattern. State is one `useState<string>` — React Compiler auto-memoizes derived results.

**Tech Stack:** Next.js 16 App Router, TypeScript, Tailwind 4, next-intl, change-case@^5.4, title-case@^4, vitest.

```ts
import {
  camelCase,
  pascalCase,
  snakeCase,
  constantCase,
  kebabCase,
  dotCase,
  pathCase,
  sentenceCase,
  noCase,
} from "change-case";
import { titleCase } from "title-case";

// Hand-written: split with noCase, then case-fold the joined words.
function lowerCase(input: string): string {
  return noCase(input).toLowerCase();
}

function upperCase(input: string): string {
  return noCase(input).toUpperCase();
}

export interface FormatDefinition {
  key: string;
  convert: (input: string) => string;
}

// Order is the canonical render order; mirrored by FORMAT_KEYS.
export const FORMATS: readonly FormatDefinition[] = [
  { key: "camelCase", convert: camelCase },
  { key: "pascalCase", convert: pascalCase },
  { key: "snakeCase", convert: snakeCase },
  { key: "constantCase", convert: constantCase },
  { key: "kebabCase", convert: kebabCase },
  { key: "dotCase", convert: dotCase },
  { key: "lowerCase", convert: lowerCase },
  { key: "upperCase", convert: upperCase },
  { key: "titleCase", convert: titleCase },
  { key: "sentenceCase", convert: sentenceCase },
  { key: "pathCase", convert: pathCase },
] as const;

export const FORMAT_KEYS: readonly string[] = FORMATS.map((f) => f.key);

export interface ConversionResult {
  key: string;
  output: string;
}

export function convertAll(input: string): ConversionResult[] {
  return FORMATS.map((f) => ({
    key: f.key,
    output: input === "" ? "" : f.convert(input),
  }));
}
```

```ts
export function detectFormats(input: string): string[] {
  if (input.trim() === "") return [];
  return FORMATS.filter((f) => f.convert(input) === input).map((f) => f.key);
}
```

```ts
describe("clipInput", () => {
```

- [ ] **Step 3: Implement `clipInput`**

Append to `libs/textcase/main.ts`:

```ts
export const MAX_INPUT_LENGTH = 1000;

export interface ClipResult {
  value: string;
  clipped: boolean;
}

export function clipInput(input: string): ClipResult {
  if (input.length <= MAX_INPUT_LENGTH) {
    return { value: input, clipped: false };
  }
  return { value: input.slice(0, MAX_INPUT_LENGTH), clipped: true };
}
```

## Notes for the implementing engineer

- **React Compiler memoization is automatic.** Do not write `useMemo`/`useCallback`/`React.memo`. Recomputation of `convertAll(input)` on every keystroke is the intended pattern.
- **Why no debounce.** Eleven function calls per keystroke on a ≤1000-char string is sub-millisecond; immediate feedback is the better UX.
- **Why round-trip detection over regex.** Each format's converter knows its own normalization rules. If `formatX(input) === input`, the input _is_ in format X by definition. Regex would duplicate (and inevitably drift from) library logic.
- **Why pin `change-case@^5.4`.** v5 is ESM-only and renamed `paramCase` → `kebabCase`. v4 imports will silently fail or produce different outputs. The edge-case lock-in tests in Task 5 will surface any future v6 behavior change.
- **Why `title-case` is a separate package.** It applies proper English capitalization rules (lowercases articles, prepositions, conjunctions) — `change-case` alone cannot do this.
- **Why no `useMemo` on `detectedSet`.** React Compiler handles it.
